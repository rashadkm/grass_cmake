DESCRIPTION

r3.in.ascii allows a user to create a (binary) GRASS 3D raster map layer from a 3D ASCII raster input file.

The tiledimension parameter defines the dimension of the tiles used in the output file. The format is: XxYxZ

The nv parameter specifies which value to convert to NULL-value. If the specified value is none, no conversion is performed. Default is none.

NOTES

A detailed description of the input format as well as the supported row/depth ordering is available in the r3.out.ascii. manual page. The order of the data in the input file does not specifiy the data order in the generated output 3D raster map which is in any case north -> south, west -> east, bottom -> top order. The data is automatically imported into the correct internal coordinate system, that is visualized in the following picture, independently from the specified ordering in the ASCII input file:

The volume coordinate system and tile layout of the imported voxel map

The version and order information in the header of the input file are not mandatory. In case no version and order information was specified, the default GRASS 6 3D ASCII format is assumed.

Note that unit tests for r3.in.ascii are implemented in the test.r3.out.ascii.sh script located in the r3.out.ascii directory.
